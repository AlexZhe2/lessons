Склонировать репозиторий
File-> New -> Project From Version Control -> Git
(в появившемся диалоговом окне указать ссылку на репозитоий)

VCS -> Git -> Clone

аналогично команде в консоле git clone ссылка_на_репозиторий

Последующая настройка проекта:
Папка с исходным кодом дб отмечена как source root
(правой кнопкой на имени проекта ->
mark directory as ->source root)

File -> Project Structure:
выбор sdk + указать папку для скомпилированных файлов

Получение изменений из удаленного репозитория:
VCS -> Git -> Pull

аналогично команде в консоле git pull

Индексация и фиксация изменений (коммит)
Красный цвет - отмечены новые файлы (непроиндексированные)
Голубой цвет - файлы с изменениями
Зеленый цвет - проиндексированные файлы

Правой кнопкой на файле или папке -> Git -> add -
для индексации изменений

аналогично команде в консоле git add ...

Правой кнопкой на файле или папке -> Git -> Commit File -
для создания коммита

аналогично команде в консоле git commit -m "message"

либо Commit & Push - для коммита и отправки изменений
в удаленный репозиторий

На случай конфликта:
1. VCS -> Git -> Stash
аналогично команде в консоле git stash
VCS -> Git -> UnStash (вернуть изменения)
аналогично команде в консоле git apply stash

2. VCS -> Git -> Reset HEAD
аналогично команде в консоле git reset --hard HEAD

git add
git commit
git pull

разрешаем конфлит

git add
git commit
git pull

Отмена изменений:
git log - просмотр истории

git stash - скрытие изменений, возвращение в состояние последнего коммита
git stash apply - возвращает скрытые изменения

git checkout <хэш сумма коммита> - можно вернуться к конкретному коммиту (только для просмотра)
git checkout master - вернуться в прежнее состояние

git checkout -b <имя новой ветки> <хэш сумма коммита> - переключаемся на желаемый коммит и начинаем работать с него, в новой созданной ветки

git reset --hard HEAD - отменить проиндексированные изменения

git reset --hard <хэш сумма коммита> - перейти к желаемому коммиту, отменив все последующие (работает в случае,если не был сделан push этого коммита)

git revert <хэш сумма коммита> - отменить опубликованные коммиты


Файл .gitignore служит для указания в нём файлов и папок, которые необходимо скрыть от системы контроля версий git
